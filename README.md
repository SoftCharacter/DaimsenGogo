# 股票供应链大屏

## 项目简介

股票供应链大屏是一个面向 A 股产业链研究场景的 AI 分析与可视化平台。用户输入一个产业主题，例如“苹果手机供应链”“宁德时代电池供应链”“华为昇腾供应链”，系统会调用大模型进行供应链拆解，生成主题、产业分类、相关上市公司及业务说明，并在看板中展示实时行情、涨跌幅、近一个月日 K 线和主题分类结构。

项目采用前后端分离架构：前端使用 React + TypeScript + Vite 构建交互式大屏，后端使用 FastAPI 提供 AI 分析、主题管理、行情代理、模型配置和历史任务接口。数据持久化以本地 JSON 文件为主，适合演示、学习、面试项目复盘和个人研究使用。

## 核心功能

### 1. AI 供应链分析

- 支持用户输入自然语言主题，自动生成供应链分析结果。
- 后端通过 ReAct / Plan-Execute 流程组织大模型分析。
- 使用 SSE 流式返回分析进度，前端可实时展示执行过程。
- 自动沉淀分析结果为“主题”，供后续看板复用。

### 2. 供应链主题看板

- 左侧展示已保存主题列表。
- 支持切换主题、保留当前主题状态、删除主题。
- 看板按供应链分类展示相关股票。
- 支持“全部视图”和分类视图，方便快速查看主题整体结构。

### 3. 股票实时行情

- 对主题下股票按代码获取实时行情。
- 现价、涨跌额、涨跌幅、开盘价、最高价、最低价和成交额来自雪球行情接口。
- 前端每 30 秒轮询一次当前主题股票行情。
- 后端对实时行情设置短缓存与并发限制，避免频繁请求外部数据源。

### 4. 近一个月日 K 线

- 每只股票展示近一个月日 K 线走势。
- K 线数据固定取最近 22 个交易日。
- 使用前复权日线数据，适合在卡片中展示近期趋势。
- 前端使用 IntersectionObserver 懒加载 K 线，只有进入视口附近的图表才会请求数据。

### 5. 模型配置管理

- 支持配置大模型 API 地址、API Key 和模型名称。
- 配置保存在本地文件中，便于演示环境快速启动。
- AI 分析前会校验模型配置是否完整。

### 6. 历史任务管理

- 保存 AI 分析任务执行记录。
- 支持查看历史分析任务，方便复盘分析过程。

## 数据来源

### 实时行情

实时行情使用雪球原始行情接口：

```text
https://stock.xueqiu.com/v5/stock/quote.json?symbol={SYMBOL}&extend=detail
```

后端会先访问雪球股票页获取必要 Cookie，再请求 quote 接口。核心字段映射如下：

| 雪球字段 | 项目字段 | 含义 |
| --- | --- | --- |
| current | current_price | 当前价格 |
| percent | change_percent | 涨跌幅 |
| chg | change | 涨跌额 |
| last_close | prev_close | 昨收价 |
| open | open_price | 开盘价 |
| high | high | 最高价 |
| low | low | 最低价 |
| amount | volume | 成交额 |
| name | name | 股票名称 |

### K 线数据

K 线数据使用 AkShare 的腾讯日线封装：

```python
ak.stock_zh_a_hist_tx(symbol="sz002460", adjust="qfq")
```

项目固定取最近 22 个交易日，作为“近一个月日 K”展示。

### 主题数据

主题数据来自 AI 分析结果并保存为本地 JSON 文件，结构包括：

- 主题 ID
- 主题名称
- 主题描述
- 供应链分类
- 每个分类下的股票列表
- 股票业务说明和供应链占比

## 使用方式

### 1. 环境准备

后端依赖 Python 环境，前端依赖 Node.js 与 npm。

本项目约定 Python 命令使用 `env_reactAgent` 环境：

```bash
conda activate env_reactAgent
```

安装后端依赖：

```bash
conda run -n env_reactAgent pip install -r backend/requirements.txt
```

安装前端依赖：

```bash
cd frontend
npm install
```

### 2. Windows 一键启动

在项目根目录执行：

```bash
scripts/start.bat
```

该脚本会通过 `scripts/launcher.py` 启动：

- 后端：`http://localhost:8000`
- 前端：`http://localhost:5173`
- API 文档：`http://localhost:8000/docs`

### 3. 手动启动后端

```bash
conda run -n env_reactAgent python -m uvicorn backend.main:app --port 8000 --host 0.0.0.0
```

### 4. 手动启动前端

```bash
cd frontend
npm run dev
```

### 5. 前端构建

```bash
cd frontend
npm run build
```

## 基本操作流程

1. 打开前端页面 `http://localhost:5173`。
2. 进入“模型配置”页面，填写 API 地址、API Key 和模型名称。
3. 进入“AI分析”页面，输入供应链主题。
4. 等待 AI 分析过程流式输出并生成主题。
5. 进入“供应链看板”，查看主题分类、股票卡片、实时行情和近一个月 K 线。
6. 后续可在左侧主题列表中切换或删除主题。

## 性能优化设计

### 1. 行情与主题解耦

主题数据只保存 AI 生成的供应链结构，不直接保存实时行情。行情进入看板后按股票代码请求，避免主题切换时重复生成分析结果。

### 2. 指定股票行情获取

实时行情不再全量下载 A 股行情，而是只请求当前主题下的股票代码，降低外部接口压力和等待时间。

### 3. 并发控制

后端对外部行情请求设置并发上限：

- 实时行情并发：5
- K 线请求并发：3

这样可以避免大量 `asyncio.to_thread` 任务挤占线程池，影响模型配置、历史任务等轻量接口。

### 4. 多级缓存

- 实时行情缓存：5 秒
- K 线缓存：1 小时
- 股票基础列表缓存：1 小时
- 本地 JSON 缓存用于外部接口异常时兜底

### 5. K 线懒加载

前端不在主题加载时一次性请求所有股票 K 线，而是在图表进入视口附近时再请求，减少首屏并发量。

### 6. 第三方输出静默

后端封装 AkShare 调用时屏蔽第三方库的进度条和下载输出，避免污染业务日志。

## 常见问题

### 1. 为什么切换主题后行情需要加载？

主题数据和行情数据是分开的。主题保存的是供应链结构，行情是实时变化数据，因此进入或切换主题后会按当前股票列表获取最新行情。

### 2. 停留在主题页面时行情会自动刷新吗？

会。前端进入主题后会立即请求一次行情，之后默认每 30 秒刷新一次当前主题股票行情。

### 3. 为什么 K 线不是一次全部加载？

看板里股票数量可能较多，如果一次性请求所有 K 线，会造成大量外部请求并影响页面响应。因此项目采用懒加载策略，用户滚动到对应股票卡片附近时才请求 K 线。

### 4. 为什么历史任务和模型配置也可能被行情请求影响？

它们虽然读取本地文件，但仍与行情请求共用同一个 FastAPI 进程和线程池资源。如果外部行情请求过多，会挤占后端处理能力。项目通过行情并发限制、缓存和懒加载降低影响。

### 5. 雪球接口返回异常怎么办？

后端会优先请求实时接口；如果失败，会尝试读取本地缓存。若没有缓存，前端对应字段可能显示 `--`。
